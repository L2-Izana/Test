2. How Two's complement is expressed

  1. Signed and Unsigned binary number system:

    For explanations and examples, let's use 8-bit number for simplicity

    As in Section 1, we know that the highest positive number can be represented is 255 (1111 1111) and people call these 0 and positive sets of numbers only are represented as a Unsigned binary number system

    But in Signed binary number system, from 255 (1111 1111) in Unsigned binary number system is only equal to -1 in this sytem. Suprised, right? Don't worry, we have just create the first ever negative number for the computer.

    The rule for the Signed binary number system is extremely easy to comprehend, just remember:

    Consider the left-most bit as a sign of the integer:

    • 0 indicates a positive number.
    • 1 indicates a negative number.

    For example: 1000 0001 indicates a negative number where as 0111 1110 indicates a postive number.

    Here, we have an important assumption from this rule: Instead of a range of 0 to 255 in Unsigned binary number system, the Signed binary number system is unable to understand all integers in range -128 to 127

  2. One's complement and Two's complement:

    As in Part a, we know that Signed binary number system can help computers understand the negative decimal numbers. So now, let's grasp the most important knowledge - Using the 2's complement number system to represent negative integers:

    Taking 7 or 0000 0111 as the example here. Let's understand the procedure of why this postive number is related to 1111 1001 or -7

    1. Firstly, we create the One's complement by toggling 1 to 0 and 0 to 1. Specifically, the One's complement of 0000 0111 is 1111 1000.

      2. Similarly, the One's complement of 0101 0101(85) is 1010 1010

    2. Secondly, we add 1 to the One's complement to create the Two's complement. Specifically, the Two's complement of 0000 1111 is 1111 1000 + 1 = 1111 1001(-7)

      3. Similarly, the Two's complement of 0101 0101 is 1010 010 + 1 = 10101011(-85).

    3. Finally, to verify:

      • 0000 0111 + 1111 1001 = 1 0000 0000

      • 0101 0101 + 1010 1011 = 1 0000 0000
